## ADDED Requirements

### Requirement: Guard на маршруты редактирования
Система ДОЛЖНА проверять наличие токена аутентификации в localStorage перед разрешением доступа к маршрутам `/edit_struct`, `/edit_workers`, `/edit_roles`. Если токен отсутствует, пользователь ДОЛЖЕН быть перенаправлен на `/catalog`.

#### Scenario: Доступ с токеном
- **WHEN** пользователь переходит на `/edit_struct` и токен есть в localStorage
- **THEN** доступ разрешён и страница загружается

#### Scenario: Доступ без токена
- **WHEN** пользователь переходит на `/edit_workers` и токена нет в localStorage
- **THEN** пользователь перенаправляется на `/catalog`

### Requirement: Глобальный route beforeEach
Vue Router ДОЛЖЕН использовать `router.beforeEach` для проверки аутентификации. Проверка ДОЛЖНА происходить перед каждой навигацией, включая прямой ввод URL в адресную строку.

#### Scenario: Прямой ввод URL
- **WHEN** пользователь вводит `/edit_roles` в адресную строку без токена
- **THEN** срабатывает guard и перенаправляет на `/catalog`

#### Scenario: Навигация через RouterLink
- **WHEN** пользователь кликает на ссылку редактирования без токена
- **THEN** навигация блокируется и пользователь остаётся на `/catalog`
